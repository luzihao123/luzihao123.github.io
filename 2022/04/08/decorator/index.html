设计模式–装饰者模式

概述

给对象动态的添加职责的方式称装饰者模式。装饰者模式能够在不改变对象自身的基础上，在程序运行期间给对象动态的添加职责。根继承相比，这是一种更灵活的方法

通过保存之前函数引用的方式改写

如果我想在某个函数上增加一些方法，但也不想改变原来函数，我可以先保存一下这个函数的引用，再新建一个函数，执行这个引用函数。

    var a = function () {
        alert(1)
    }
    var _a = a
    a = function () {
        _a()
        alert(2)
    }

这样就可以在a函数上增加新方法而不修改原来的a函数，但这样的缺陷也存在，

  1. 我们需要维护中间变量_a, 如果所加的装饰函数不止1个，会有很多中间变量需要维护
  2. 会存在this被劫持的问题
     // 错误示例
var _getElement = document.getElementById
document.getElementById = function(id) {
    alert(1)
    return _getElement(id)
}
var button = document.getElementById('button')
    上面方法是错误的，因为_getElement()成了一个全局函数，而我们希望的document.getElementById 的this引用指向document，所以需要改写return方法
    return _getElement.call(document, ...arguments)

AOP

在Function原型上定义两个方法，before,after, 然后我们用这两个方法来给原函数添加功能。需要先保留当前函数的this指向，在before,after的返回函数中使用正确的this指向，以免this指向了全局

  Function.prototype.before = function(beforeFn) {
      var _self = this
      return function() {
          beforeFn.apply(this, arguments)
          return _self.apply(this, arguments)
      }
  }
  Function.prototype.after = function(afterFn) {
      var _self = this
      return function () {
          var ret = _self.apply(this, arguments)
          afterFn.apply(this, arguments)
          return ret
      }
  }

  // 使用
  a = a.before(() => alert(1)).aflter(() => alert(2))

有了before和after方法，就可以形成一条装饰链，在函数前后加入各种功能，处理各种实际问题，比如在ajax请求前加入一些参数，在表单提交前进行验证
