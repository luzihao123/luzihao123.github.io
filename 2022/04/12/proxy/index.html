设计模式-代理模式

概述

给目标对象A提供一个中间对象B，如果需要访问目标对象A，得通过这个中间对象B来进行。有两种实践中的影子。1.保护代理，代理可以先过滤掉一些访问 2.虚拟代理，代理把一些开销很大的操作延迟到真正需要它的时候去创建 3 也可以把访问或修改对象的逻辑放在代理层处理

图片加载的代理

在图片加载时，需要加一个loading出现在图片显示之前，那么可以用这个方法

    var myImage = (function() {
        var imgNode = document.createElement('img')
        document.appendChild(imgNode)
        return {
            setSrc:function(src) {
                imgNode.src = src
            }
        }
    })()

    var ProxyImage = (function() {
        var img = new Image()
        img.onload = function() {
            myImage.setSrc(img.src)
        }
        return {
            setSrc: function(src) {
                myImage.setSrc('../loading.gif')
                img.src = src
            }
        }
    })()

    ProxyImg.setSrc('xxx')

这样做的好处是符合了 单一职责原则 和 开放封闭原则 , 我们的myImage只负责给图片节点设置src，这符合了单一职责原则，我们也没有改变myImage原有的功能，而是把加载中图片设置的功能放在了另一个函数，这符合开放封闭原则

es6的proxy

es6 中引入了Proxy，可以通过Proxy 对对象访问和修改进行拦截
proxy 接收两个参数, target 和 handler, handler是拦截的具体方法，可以拦截对象原型上的几个方法，用的最多的是get，set，
另外还有deleteProperty， ownKeys等
所以用了Proxy后我就只能用赋值的方法来传入，这样才能被set方法拦截
图片加载的例子用proxy重写一下如下

    let proxyImg = new Proxy(myImage, {
        set(target, propKey, value, receiver) {
            let img = new Image
            img.onload = function () {
                myImage.setSrc(value)
            }
            myImage.setSrc('d:\\cs\\sj\\1.png')
            img.src = value
        }
    })
    proxyImg.setSrc = 'xxx'
