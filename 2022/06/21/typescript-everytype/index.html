typescript–基本类型

本节记录文档中基础的类型定义

原始类型 string number boolean

数组类型

定义语法: number[] 或者 Array
需注意: [number] 这是元组定义，与数组不同

联合类型

由两种或以上类型组合成的类型，使用 ｜ 符号， 表示可以是这几种类型中任何的一种。
提供符合联合类型的值是很容易的，只需要是其中一种类型。但在使用联合类型时，ts要求使用方法必须符合联合类型中每一个类型，比如
我定义一个number和string的联合类型，我想直接使用一些string特有的方法，那是不行的，解决方法是收窄类型。
当ts能用类型推断得出更小类型时，就会发生收缩

    function printId(id: number | string) {
        if (typeof id === "string") {
            // In this branch, id is of type 'string'
            console.log(id.toUpperCase());
        } else {
            // Here, id is of type 'number'
            console.log(id);
        }
    }

ts 会在判断时自动收缩类型范围，第一个if进入时id类型为string，在else中时，id类型已经只能是number

类型别名 (type aliases) 和 接口 (interfaces)

类型别名： 可以是任何类型的别名，用来保存一个类型的引用， (可以是一个对象的类型，或是一个联合类型)
接口： 命名对象类型的方法

都定义对象类型时的区别: type 继承时候使用 & ， interface 继承时用 extends
type 不允许同一个名称重复定义， interface 可以同一个名称重复定义，会把所有属性合并成一个

类型断言 assertion

编码者清楚知道一个值的具体类型时，可以用 as 语法给类型断言

   const myCanvas = document.getElementById("main_canvas") as HTMLCanvasElement;

类型字面量 literal type

定义类型为特定的字符或者数字，当只定义一个值时是没什么用的，一般和联合类型配合使用。

    const alignment = 'left' | 'center' | 'right'

非空断言 ！ 可选属性 ？

    type obj = { first: string; last?: string }

    function liveDangerously(x?: number | null) {
        // No error
        console.log(x!.toFixed());
    }
